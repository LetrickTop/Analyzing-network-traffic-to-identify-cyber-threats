Основы работы

Система анализа сетевого трафика позволяет:

  • Захватывать сетевые пакеты в реальном времени
  • Анализировать трафик на предмет аномалий
  • Визуализировать статистику по протоколам и сервисам
  • Хранить историю трафика для последующего анализа

Для начала работы выберите нужный раздел в верхнем меню.

Сниффинг трафика

1Запуск сниффинга

Для начала захвата трафика:

  1. Перейдите в раздел "Сниффинг трафика"
  2. Нажмите кнопку
  3. Дождитесь появления первых пакетов в таблице
Для работы сниффера требуются права администратора на вашей системе.
2Анализ данных

В таблице отображаются следующие параметры:

Параметр Описание
Время Метка времени пакета
Источник/Назначение IP-адреса отправителя и получателя
Протокол Используемый сетевой протокол (TCP, UDP, ICMP)
Размер Размер пакета в байтах
Сервис Сетевой сервис (HTTP, FTP, SSH и т.д.)
3Остановка сниффинга

Для остановки захвата трафика:

  1. Нажмите кнопку
  2. Дождитесь завершения обработки последних пакетов
Длительный захват трафика может потребовать значительных ресурсов системы.

Анализ кибератак

1Запуск анализа

Для проверки трафика на атаки:

  1. Перейдите в раздел "Анализ кибератак"
  2. Нажмите кнопку
  3. Дождитесь результатов анализа

Система использует машинное обучение для выявления:

  • DDoS атак
  • Попыток сканирования портов
  • Подозрительных флагов TCP
  • Аномальных объемов трафика
2Интерпретация результатов
Нормальный трафик

Зеленый индикатор означает, что трафик соответствует нормальным паттернам.

Обнаружена атака!

Красный индикатор сигнализирует о потенциальной атаке. Проверьте детали для получения дополнительной информации.

Вероятность атаки отображается в процентах. Значения выше 70% следует рассматривать как серьезную угрозу.

Аналитика трафика

1Использование аналитики

В разделе "Аналитика" представлены:

  • Круговая диаграмма распределения протоколов
  • Гистограмма использования сетевых сервисов
  • Статистика по количеству пакетов

Данные обновляются автоматически при поступлении нового трафика.

Используйте аналитику для выявления аномальных пропорций трафика, которые могут указывать на атаку.

Часто задаваемые вопросы

Для захвата сетевого трафика требуются права администратора (root/sudo на Linux/macOS или запуск от имени администратора на Windows).
Данные в таблице обновляются каждую секунду. Графики аналитики обновляются при изменении данных.
Система обучена обнаруживать: DoS/DDoS атаки, сканирование портов, попытки brute-force, подозрительные флаги TCP, аномальную активность сервисов.
